# Embedded Knowledge Base Plan


# 1. 日常执行计划


每天投入：

10~15分钟。


目标：

持续积累工程经验。


---

# 每日流程


## 第一步：选择记录内容（3分钟）


回顾当天：

是否遇到：

- Bug
- 调试问题
- 新技术
- 新工具
- 代码设计问题


选择一个。


---

## 第二步：整理记录（10分钟）


使用模板：

```md
# 标题


## 背景


## 问题


## 分析过程


## 解决方案


## 总结
```


---

## 第三步：归档（2分钟）


根据类型保存：

驱动：

Driver/


Bug：

Debug/


项目：

Projects/


协议：

Protocol/

# 5. 不建议记录

- 复制Datasheet

# 6. 判断标准


好的记录：


```text
STM32 UART DMA偶发丢数据

现象：

接收数据偶尔错误。


分析：

DMA缓存处理异常。


定位：

查看DMA状态和缓存。


解决：

修改缓存刷新流程。
```


不好的记录：


```text
UART是什么

STM32是什么
```
