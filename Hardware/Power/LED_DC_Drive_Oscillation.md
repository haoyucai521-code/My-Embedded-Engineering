# LED DC 直驱导致电源振荡与亮度呼吸效应

> 文档日期：2026-07-21

---

## 先搞懂两个概念

**DC 直驱**：LED 一直亮，不关。4 个灯就是 4 个同时亮到底。

```
LED1: ████████████████  一直亮
LED2: ████████████████  一直亮
LED3: ████████████████  一直亮
LED4: ████████████████  一直亮
       同一时刻 4 个灯都在吃电流
```

**TDM 轮转**：LED 轮流亮，每次只亮一个。切得够快人眼看不出闪烁。

```
LED1: ██________________  亮一小段
LED2: ___██______________  轮到它亮
LED3: ______██___________  轮到它亮
LED4: _________██________  轮到它亮
       同一时刻只有 1 个灯在吃电流
```

---

## 现象

LED 常量模式下，亮度周期性波动——亮→暗→亮→暗，像在呼吸。

闪烁模式下不可见（亮灭切换太快，盖过了波动）。

---

## 根因

代码里有一条"优化"分支：当所有 LED 颜色相同时，跳过轮转，一次性全开。本意是少写几次 GPIO 寄存器，看起来更高效。

结果走了 DC 直驱，4 个 LED 同时吃电流，峰值电流是 TDM 的 4 倍。

**为什么电流大了会振荡：**

GPIO 的电来自板子上的 3.3V 供电。供电不是理想电压源，走线、引脚、内部 bond 线都有电阻。根据欧姆定律，电流流过电阻就产生压降。

```
电流大 → 走线压降大 → GPIO 实际电压低于 3.3V → LED 变暗
LED 变暗 → 等效电流变小 → 走线压降减小 → 电压恢复
电压恢复 → LED 变亮 → 电流又变大 → 电压又跌 → 循环
```

这是一个正反馈环，没法自己收敛到稳态，只能在两个状态之间来回跳。人眼看到的就是亮暗循环。

**为什么 TDM 没事：**

同一时刻只有 1 路 LED 亮，峰值电流是 DC 的 1/4。供电顶得住，不跌就不振荡。

**验证逻辑：**

- 唯一的变量是扫描方式（DC ↔ TDM），没改硬件、没改 GPIO、没改 LED
- 切 DC 必现呼吸效应，切 TDM 必消失
- 结论：根因是 DC 直驱的峰值电流超过了供电系统的承受能力

---

## 修复

去掉 DC 直驱分支，统一走 TDM 轮转：

```
每个 tick (2ms):
  1. 关中断 → 拷贝 LED 状态快照 → 开中断
  2. 全关所有通路（消隐，防残影）
  3. 当前槽位的 LED 活跃 → 点亮
  4. 槽位 +1，下一轮
```

| 对比 | DC 直驱 | TDM 轮转 |
|------|---------|----------|
| 同时亮的 LED | 4 个 | 1 个 |
| 峰值电流 | 高 | 低（1/4） |
| 电源稳定性 | 振荡 | 稳定 |
| 代码分支 | 3 个 | 1 个 |

4 路 × 2ms = 8ms 一个周期 = 125Hz，远超 50Hz 人眼闪烁阈值，看不出闪。

**风险**：25% 占空比理论亮度低于 100% DC，但人眼积分效应下实际亮度足够，已验证。

---

## 结论

**LED 多路驱动不要用 DC 直驱，一律走 TDM。**

DC 直驱看着省事，但峰值电流是 TDM 的 N 倍。供电系统有内阻是物理事实，电流上去电压必然跌，跌了 LED 就暗，暗了电流就减，减了电压就回弹——这个振荡绕不开。TDM 用时间换电流，硬件代价为零。
